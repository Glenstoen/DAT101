1. What is the difference between invoking and creating a function?

    2. Creating a function defines its "assignment" and tells it what to do. invoking a function calls on that function to do its assigned task.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are used to "finish" a statement, and codeblocks are several lines of code within a block contained by curly brackets : {}

  3. What happens when we try to store values from a function when it doesn't return any?

    4. it will then store the values as undefined

  4. How do we separate parameters from each other when creating a function?

    5. when creating the function, we add the parameters inside the parentheses at the end, seperated by a comma

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. The parameter names in the fuction are basically placeholders, when invoking the function, the names are replaced by the actual value. but the order of input matters, because the names are referenced in the function as variables that store the actual value input coming from the parameters

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. the function is created to use a set number of parameters to do its assigned task, if a parameter is missing, the function can not do its task properly so the parameter becomes undefined.

  7. Explain what happens when we use recursive functions.

    8. a recursive function will keep invoking itself until it reaches a set stop condition
